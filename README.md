#### 1. 소개
이 프로젝트는 언리얼 엔진 멀티플레이 네트워크를 학습하기 위한 숫자 야구 게임 입니다.
두 명의 플레이어가 1~9 사이의 중복되지 않는 3자리 숫자를 입력하고,
서버가 생성한 정답과 비교하여 Strike / Ball / Out 결과를 반환합니다.

+ 먼저 3 Strike 를 달성한 플레이어가 승리
+ 두 플레이어가 모두 3번의 기회를 사용하면 무승부
+ 승리 또는 무승부 발생 시 게임 자동 리셋

멀티플레이 환경에서 GameMode, PlayerState, RPC, Replication, Widget 업데이트를 자연스럽게 익히도록 설계되었습니다.

------------
#### 2. 게임 규칙

*게임 규칙*

기본 규칙
+ 서버는 1~9 사이의 중복 없는 3자리 숫자 생성 (예: 386, 594)
+ 플레이어는 “123”, “743” 등 3자리 중복 없는 숫자 입력

*판정 기준*
+Strike(S): 위치 + 숫자 동일
+Ball(B): 숫자는 같지만 위치 다름
+둘 다 아니면 OUT

*예시*
정답	입력	결과
386	127	OUT
386	167	0S1B
386	367	1S1B
386	396	2S0B
386	386	3S0B → 승리

*승리 조건*
3회 안에 3S 달성한 플레이어

*무승부 조건*
두 플레이어 모두 3번의 기회를 소모했으나 3S 실패

----------------
#### 3. 프로젝트 구조

Source/

 ├── GameMode/             정답 생성, 승리/무승부 판정, 게임 리셋

 ├── PlayerController/     입력 처리, Server RPC, UI 업데이트

 ├── PlayerState/          시도 횟수 관리 (Replication)

 ├── Widgets/              채팅/결과/공지 위젯

 ---------------------
#### 4. 핵심 기능

 *핵심 기능*
1. 서버 정답 생성
+ 1~9 사이 중복 없는 3자리 숫자 랜덤 생성
+ 게임 시작 시 GameMode에서 자동 실행

2. 플레이어 입력 검증
+ 정확히 3자리인지
+ 숫자로만 이루어졌는지
+ 중복이 없는지

검증 실패 시 오류 메시지 반환.

3. 숫자 야구 판정
+ 입력된 숫자와 정답 비교 후
+ XSYB, OUT 문자열 생성

4. PlayerState 기반 시도 횟수 관리
+ 최대 시도 횟수: 3
+ 입력할 때마다 현재 시도 횟수 증가
+ UI에 현재/최대 표시

5. 승리/무승부/게임 리셋
+ GameMode가 전체 흐름 통제
+ 위젯으로 결과 공지
+ 새로운 정답 생성 후 게임 재시작

------------------
#### 5. 게임 진행 흐름
1. 서버 → 정답 숫자 생성
2. 플레이어 → 숫자 입력
3. 서버 → 판정 후 결과 전달
4. PlayerState → 시도 횟수 증가
5. 승리 or 무승부 판정
6. 결과 위젯 표시
7. 게임 전체 리셋 후 재시작
